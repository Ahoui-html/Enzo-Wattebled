Goals

Voter is a vote application to ease intern decisions for big companies. The application had to manage a large number of participants while being user friendly (our typical user is not very familiar with technologies).


Challenges

First: implement the basic functions of a voting application: define the date, time, add additional information, create different types of votes (MCQ; radio; split vote, list election)

Secondly: Add new functionalities in this domain: such as being able to give one's vote to another participant, or enable manual vote counting to add an additional level of security on certain votes (useful for large companies).

To be a relevant application that fits in our current professional context, we had to take into consideration the arrival of participants (in person or remotely). To control and ensure a maximum participation rate for each vote, we have set up a communication part to follow up with the participants by email or SMS.


Design method and tools

Audit existing vote and poll application, listing key points in the user flow that work well or could be improved. Using these key points to build our own experience, and in parallel, integrate those previous features.

Wireframing all functionalities, then apply graphic charter.
